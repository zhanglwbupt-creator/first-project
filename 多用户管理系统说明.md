# 多用户管理系统使用说明

## ✨ 功能概述

已成功添加完整的用户认证和多用户管理系统，支持：
- ✅ 用户注册/登录
- ✅ JWT Token认证
- ✅ 密码加密存储（bcrypt）
- ✅ 多用户数据隔离
- ✅ 自动路由守卫
- ✅ Token自动管理

---

## 🎯 用户操作流程

### 1. 首次访问
- 打开应用会自动跳转到登录页面
- 可以选择"登录"或"注册"

### 2. 注册新用户
1. 点击"注册"标签
2. 填写用户名（至少3个字符）
3. 填写密码（至少6个字符）
4. 可选填写昵称和邮箱
5. 点击"注册"按钮
6. 注册成功后自动登录并跳转到首页

### 3. 登录
1. 输入用户名和密码
2. 点击"登录"按钮
3. 登录成功后跳转到首页

### 4. 退出登录
- 清除浏览器localStorage中的token即可
- 刷新页面会跳转到登录页

---

## 🔒 数据安全机制

### 1. 密码加密
- 使用bcrypt算法加密存储
- 密码强度：10轮哈希
- 数据库中不存储明文密码

### 2. Token认证
- 使用JWT (JSON Web Token)
- Token有效期：7天
- 每次请求自动携带Token

### 3. 数据隔离
每个用户只能看到和操作自己的数据：
- 词库（word_banks）- 按user_id隔离
- 单词（words）- 按user_id隔离
- 学习记录（study_records）- 按user_id隔离

## 🗄️ 数据库结构

### users表（用户表）
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,      -- 用户名（唯一）
  password TEXT NOT NULL,              -- 加密密码
  nickname TEXT,                       -- 昵称
  email TEXT,                          -- 邮箱
  created_at DATETIME,                 -- 创建时间
  updated_at DATETIME                  -- 更新时间
)
```

### 数据表关联
所有业务表都添加了user_id字段：
- `word_banks.user_id` → 关联用户
- `words.user_id` → 关联用户
- `study_records.user_id` → 关联用户

## 🔧 API接口

### 认证相关

#### 1. 用户注册
```
POST /api/auth/register
Body: {
  "username": "testuser",
  "password": "123456",
  "nickname": "测试用户",
  "email": "test@example.com"
}
Response: {
  "message": "注册成功",
  "token": "eyJhbGci...",
  "user": {
    "id": 1,
    "username": "testuser",
    "nickname": "测试用户",
    "email": "test@example.com"
  }
}
```

#### 2. 用户登录
```
POST /api/auth/login
Body: {
  "username": "testuser",
  "password": "123456"
}
Response: {
  "message": "登录成功",
  "token": "eyJhbGci...",
  "user": {...}
}
```

#### 3. 获取用户信息
```
GET /api/auth/profile
Headers: Authorization: Bearer {token}
Response: {
  "user": {...}
}
```

### 业务接口（需要认证）

所有词库、单词、学习相关的API都需要在请求头中携带Token：
```
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

包括：
- `GET/POST /api/wordbanks/*` - 词库管理
- `GET/POST /api/study/*` - 学习功能

## 🛡️ 路由守卫

### 保护的路由（需要登录）
- `/` - 首页
- `/wordbank` - 词库列表
- `/wordbank/detail` - 词库详情
- `/import` - 单词导入
- `/study/*` - 所有学习相关页面

### 公开的路由（无需登录）
- `/auth` - 登录/注册页面

### 自动跳转逻辑
1. 未登录访问受保护页面 → 自动跳转到 `/auth`
2. 已登录访问 `/auth` → 自动跳转到 `/`
3. Token过期或无效 → 清除token并跳转到 `/auth`

## 📦 技术实现

### 后端技术栈
- **jsonwebtoken** - JWT token生成和验证
- **bcryptjs** - 密码加密
- **Express中间件** - 认证拦截

### 前端技术栈
- **Vue Router守卫** - 路由权限控制
- **Axios拦截器** - 自动添加Token、处理401错误
- **localStorage** - 存储Token和用户信息

### 关键文件

#### 后端
- `backend/routes/auth.js` - 认证路由
- `backend/database.js` - 用户数据操作
- `backend/server.js` - 注册认证中间件

#### 前端
- `src/views/Auth.vue` - 登录/注册页面
- `src/router/index.js` - 路由配置和守卫
- `src/utils/axios.js` - Axios拦截器
- `src/main.js` - 导入axios拦截器

## 🔐 安全建议

### 生产环境配置

1. **修改JWT密钥**
```env
# .env文件
JWT_SECRET=your-super-secret-key-here
```

2. **启用HTTPS**
- 所有API请求使用HTTPS
- 防止Token被窃听

3. **Token刷新机制**
- 实现refresh token
- 避免频繁重新登录

4. **密码策略**
- 最少8个字符
- 包含大小写字母、数字、特殊字符
- 常见密码检查

5. **防暴力破解**
- 登录失败次数限制
- IP封禁机制
- 验证码

## 📝 注意事项

### 1. 旧数据处理
- 旧数据库没有user_id字段
- 已删除旧数据库重新初始化
- 如需迁移旧数据，需要手动添加user_id

### 2. Token存储
- Token存储在localStorage
- 容易受到XSS攻击
- 生产环境建议使用HttpOnly Cookie

### 3. 并发控制
- 同一用户可以在多设备登录
- 每个设备的token独立
- 暂不支持踢出其他设备

### 4. 数据备份
- 定期备份数据库
- 导出用户数据功能（待实现）
