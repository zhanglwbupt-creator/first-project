# 版本升级说明 - v1.1 → v1.2

> **项目名称：** 中小学生背单词小程序  
> **升级日期：** 2026-04-11  
> **升级类型：** 重大策略调整

---

## 📌 核心变化

### 开发策略调整

**v1.1（原方案）：**
```
直接开发微信小程序
技术栈：uni-app + 微信云开发
周期：4周
```

**v1.2（新方案）：**
```
先开发竖屏网页版（H5） → 功能完善 → 移植到小程序
网页版技术栈：Vue3 + Vite + Vant 4 + Node.js + Express
小程序技术栈：uni-app + 微信云开发（第二阶段）
周期：网页版4周 + 小程序移植2-3周
```

---

## 🎯 为什么要调整？

### 原方案的问题
1. ❌ 微信审核流程长，迭代慢
2. ❌ 调试不够方便（微信开发者工具限制）
3. ❌ 前期需要企业资质注册小程序
4. ❌ 功能验证成本高

### 新方案的优势
1. ✅ **开发速度快** - 无需审核，实时预览，热更新
2. ✅ **调试方便** - 浏览器DevTools，CSS调试直观
3. ✅ **零成本启动** - Vercel/Netlify免费部署
4. ✅ **快速验证** - 上线快，收集用户反馈
5. ✅ **代码复用** - 业务逻辑80%+可复用到小程序
6. ✅ **降低风险** - 先验证功能，再投入小程序开发

技术栈保持不变，但会基于网页版代码进行适配。

## 💡 功能实现差异

### OCR识别

| 维度 | 网页版（Tesseract.js） | 小程序（微信OCR） |
|------|----------------------|------------------|
| 识别速度 | 3-8秒 | 1-3秒 |
| 识别精度 | 85-95% | 90-98% |
| 首次加载 | 10MB模型 | 无需加载 |
| 费用 | 免费 | 有免费额度 |
| 离线支持 | 支持（模型本地） | 不支持 |

### 消息提醒

| 维度 | 网页版（浏览器通知） | 小程序（订阅消息） |
|------|---------------------|-------------------|
| 稳定性 | 中（依赖浏览器） | 高（微信保证） |
| 用户授权 | 需要手动授权 | 需要手动授权 |
| 发送限制 | 无限制 | 每月3次/用户 |
| iOS支持 | 有限 | 完全支持 |
| 后台运行 | Service Worker | 微信后台 |

### 数据存储

| 维度 | 网页版 | 小程序 |
|------|--------|--------|
| 本地存储 | localStorage（5MB） | wx.storage（10MB） |
| 云端存储 | MySQL | 云数据库 |
| 同步机制 | 需要自建API | 云开发自动同步 |
| 离线访问 | PWA支持 | 原生支持 |

## 🚀 迁移策略（网页版 → 小程序）

### 可复用部分（80%+）
- ✅ 业务逻辑（学习算法、复习算法）
- ✅ 状态管理（Pinia stores）
- ✅ API接口设计
- ✅ UI组件（需适配为小程序组件）

### 需要重写部分（20%）
- ❌ 登录系统（本地配置 → 微信登录）
- ❌ OCR模块（Tesseract.js → 微信OCR插件）
- ❌ 通知系统（Notification API → 订阅消息）
- ❌ 存储API（localStorage → wx.storage）
- ❌ 后端服务（Express → 云函数）

### 迁移步骤
1. 创建uni-app项目
2. 复用业务逻辑代码
3. 适配UI组件
4. 替换平台特定API
5. 集成微信能力
6. 测试优化
7. 提交审核

## 🎓 经验总结

### 为什么要网页版先行？

1. **快速验证胜过完美规划**
   - 与其花4周开发小程序，不如2周上线网页版验证核心功能
   - 用户反馈比任何规划都重要

2. **降低试错成本**
   - 网页版开发成本低，修改灵活
   - 发现方向错误可以快速调整

3. **技术债务可控**
   - 网页版可以专注业务逻辑，不受平台限制
   - 移植时再考虑平台特定优化

4. **团队能力提升**
   - 网页版开发过程中熟悉业务逻辑
   - 移植到小程序时更有针对性

## 📞 常见问题

### Q1: 总开发周期变长了吗？
**A:** 是的，总周期从4周延长到6-7周。但网页版4周后即可上线使用，小程序是增量开发。

### Q2: 代码复用率真的能达到80%吗？
**A:** 业务逻辑（算法、状态管理）可以100%复用，UI组件需要适配，整体约70-80%。

### Q3: 网页版用户体验会比小程序差吗？
**A:** 初期可能略有差距（如通知功能），但PWA技术可以弥补大部分差异。核心学习体验一致。

### Q4: 是否需要重新设计数据库？
**A:** 不需要。网页版使用MySQL，小程序使用云数据库，数据结构保持一致。

### Q5: 百度翻译API密钥如何保护？
**A:** 网页版通过后端Express代理调用，密钥存储在后端环境变量中，不暴露给前端。
