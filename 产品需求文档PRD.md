## 1. 产品概述

### 1.1 产品背景

随着英语教育从小学阶段开始普及，中小学生对单词记忆工具的需求日益增长。传统的背单词方式枯燥乏味，难以坚持。本产品旨在为中小学生提供一款**有趣、高效、便捷**的背单词小程序。

### 1.2 产品定位

一款专为中小学生设计的**轻量级背单词工具**，支持通过图片/拍照导入词库，参考百词斩的学习模式，帮助学生科学记忆单词。

### 1.4 技术方案（已确认）

#### 第一阶段：竖屏网页版（H5）

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | Vue3 + Vite | 快速开发，热更新，调试方便 |
| **UI框架** | Vant 4 (移动端组件库) | 专为移动端设计，组件丰富 |
| **状态管理** | Pinia | Vue3官方推荐，轻量高效 |
| **路由** | Vue Router 4 | 单页应用路由管理 |
| **后端服务** | Node.js + Express | 轻量级后端，快速开发 |
| **数据库** | SQLite / MySQL | 本地开发用SQLite，生产用MySQL |
| **OCR识别** | Tesseract.js | 浏览器端OCR，无需后端 |
| **词典API** | 百度翻译API | 翻译准确，支持例句 |
| **部署** | Vercel / Netlify | 免费部署，自动HTTPS |

#### 第二阶段：微信小程序移植

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | uni-app (Vue3) | 从网页版移植，代码复用率70%+ |
| **后端服务** | 微信云开发 | 免运维，与微信生态集成 |
| **OCR识别** | 微信OCR插件 | 原生支持，识别率高 |
| **登录方式** | 微信一键登录 | openid认证 |
| **部署** | 微信小程序平台 | 微信审核发布 |

## 3. 功能模块设计

### 3.1 词库管理模块

#### 3.1.1 图片/拍照/Excel导入词库

**网页版实现：**
- ✅ 支持拍照或选择相册图片（`<input type="file" capture>`）
- ✅ 使用Tesseract.js进行浏览器端OCR识别
- ✅ 支持Excel导入词库（SheetJS库）
- ✅ 支持常见单词列表格式：单词-释义-例句
- ✅ 支持手动编辑和修正识别结果

**小程序版实现（后续）：**
- ✅ 支持通过微信拍照或选择相册图片
- ✅ 使用微信OCR插件识别文字
- ✅ 支持Excel导入词库
- ✅ 支持手动编辑和修正识别结果

#### 3.1.2 多词库管理

- ✅ 支持创建多个词库（如：小学英语、初中词汇、中考词汇等）
- ✅ 词库支持分组（如：人教版、外研版等）
- ✅ 支持词库内单词的增删改查

### 3.2 学习模块

#### 3.2.1 学习模式（参考百词斩）

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 📖 **看词识意** | 显示单词，选择正确释义 | 快速记忆 |
| 🖼️ **看图选词** | 显示图片，从四个单词中选择正确选项 | 图像记忆 |
| ✍️ **拼写练习** | 输入单词进行拼写练习 | 强化记忆 |
| 🎧 **听音辨词** | 播放单词发音，选择正确单词 | 听力训练 |

#### 3.2.2 每日学习计划

- ✅ 用户可设置每日学习单词数量（**默认10个/天**）
- ✅ 系统按天滚动推送新单词
- ✅ 完成今日学习后显示学习报告

### 3.3 复习模块

#### 3.3.1 艾宾浩斯遗忘曲线复习

根据记忆规律自动安排复习时间点：

| 复习轮次 | 时间间隔 |
|---------|---------|
| 第1次复习 | 1天后 |
| 第2次复习 | 2天后 |
| 第3次复习 | 4天后 |
| 第4次复习 | 7天后 |
| 第5次复习 | 15天后 |

- ✅ 用户可设置每日复习单词数量上限

#### 3.3.2 复习提醒

**网页版实现：**
- ✅ 支持浏览器通知提醒（Notification API）
- ✅ 用户可设置提醒时间（如：每天晚上8点）
- ✅ Service Worker后台提醒（PWA支持）

**小程序版实现（后续）：**
- ✅ 支持微信订阅消息提醒
- ✅ 用户可设置提醒时间

### 3.4 词典查询模块

#### 3.4.1 互联网词典集成

- ✅ 导入词库时自动查询缺失释义
- ✅ 支持有道词典API/百度翻译API
- ✅ 查询内容包括：音标、释义、例句、发音

### 3.5 后台管理系统

#### 3.5.1 词库管理

- ✅ 管理员可批量导入/导出词库
- ✅ 支持词库审核和发布
- ✅ 支持预设常用词库模板

#### 3.5.2 数据统计

- ✅ 用户学习数据汇总（如用户数、学习人数等）

## 5. 数据字典

### 5.1 核心数据表

#### 词库表 (word_bank)

| 字段名 | 数据类型 | 必填 | 说明 |
|--------|---------|------|------|
| bank_id | INT | 是 | 词库ID 主键自增 |
| bank_name | VARCHAR(100) | 是 | 词库名称 |
| bank_group | VARCHAR(50) | 否 | 词库分组 如：小学、初中 |
| description | TEXT | 否 | 词库描述 |
| word_count | INT | 是 | 单词数量 默认0 |
| status | TINYINT | 是 | 状态 0-草稿 1-已发布 |

#### 单词表 (word)

| 字段名 | 数据类型 | 必填 | 说明 |
|--------|---------|------|------|
| word_id | INT | 是 | 单词ID 主键自增 |
| bank_id | INT | 是 | 词库ID 外键关联word_bank |
| word_text | VARCHAR(100) | 是 | 单词 |
| phonetic | VARCHAR(100) | 否 | 音标 |
| definition_en | TEXT | 否 | 英文释义 |
| definition_cn | TEXT | 否 | 中文释义 |
| example_sentence | TEXT | 否 | 例句 |
| audio_url | VARCHAR(500) | 否 | 发音音频URL |

#### 学习记录表 (study_record)

| 字段名 | 数据类型 | 必填 | 说明 |
|--------|---------|------|------|
| record_id | INT | 是 | 记录ID 主键自增 |
| openid | VARCHAR(100) | 是 | 用户OpenID |
| word_id | INT | 是 | 单词ID 外键关联word |
| learn_date | DATE | 是 | 学习日期 |
| learn_status | TINYINT | 是 | 学习状态 0-未学 1-学习中 2-已掌握 |
| next_review_date | DATE | 否 | 下次复习日期 |

## ✅ 已确认的关键决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| **数据存储** | 本地为主+云端同步 | 兼顾离线可用和多设备同步，降低成本 |
| **登录方式** | 微信一键登录（本地JWT） | 降低使用门槛，网页版先用JWT |
| **目标用户** | 学生自主使用 | 简化产品逻辑，专注核心学习体验 |
| **词典API** | 百度翻译API + 有道备用 | 翻译准确，例句丰富，稳定性好 |
| **开发框架** | Vue3 + Vite + Express | 跨平台能力，开发效率高，生态完善 |
| **后端服务** | Node.js + SQLite/MySQL | 免运维，与微信生态深度集成 |
| **词库来源** | 内置+自定义 | 降低使用门槛，同时保持灵活性 |

### 7.3 核心架构决策

#### 7.3.1 数据存储策略
```
本地存储（localStorage）
├── 用户Token（JWT）
├── 词库配置
└── 临时缓存

后端数据库（SQLite/MySQL）
├── users（用户表）
├── word_banks（词库表）
├── words（单词表）
└── study_records（学习记录表）
    ├── study_mode（choice/spell）
    ├── mastery_level（0/1/2）
    └── review_stage（复习阶段）
```

**设计优势：**
-  离线可用（本地缓存配置）
-  多用户支持（独立数据隔离）
-  性能优化（减少网络请求）
-  灵活扩展（SQLite→MySQL无缝切换）

#### 7.3.2 学习记录模型
```javascript
study_records {
  id: INTEGER PRIMARY KEY,
  user_id: INTEGER,
  word_id: INTEGER,
  bank_id: INTEGER,
  study_date: DATE,           // 学习日期
  study_type: TEXT,           // learn/review（学习/复习）
  study_mode: TEXT,           // choice/spell（看词识意/拼写）
  status: INTEGER,            // 0未掌握 1模糊 2掌握
  correct: INTEGER,           // 是否正确
  review_stage: INTEGER,      // 复习阶段（0-5）
  next_review_date: DATE,     // 下次复习日期
  mastery_level: INTEGER      // 掌握度（0陌生/1模糊/2认识）
}
```

**核心逻辑：**
- 同一单词在不同模式下有独立的学习记录
- 复习列表查询不去重，保留所有模式的错误记录
- 艾宾浩斯算法根据mastery_level和review_stage计算下次复习时间
