## 🔧 手动部署

### 步骤1：上传文件

#### 方法A：使用SCP命令
```powershell
# 在PowerShell中执行
scp wordbook-deploy.tar.gz root@82.156.202.245:/root/
```

#### 方法B：使用WinSCP（图形界面）
1. 下载安装 WinSCP：https://winscp.net/
2. 连接到服务器：
   - 主机：82.156.202.245
   - 用户名：root
   - 密码：（你的服务器密码）
3. 将 `wordbook-deploy.tar.gz` 上传到 `/root/`

### 步骤2：SSH连接云主机

```bash
ssh root@82.156.202.245
# 输入密码
```

### 步骤3：检查是否已安装Node.js

```bash
node -v
npm -v
```

**如果未安装Node.js**，执行以下命令：

#### CentOS/RHEL/Alibaba Cloud Linux：
```bash
# 安装Node.js 18.x
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs

# 验证
node -v  # 应该显示 v18.x.x
npm -v   # 应该显示 9.x.x 或更高
```

#### Ubuntu/Debian：
```bash
# 安装Node.js 18.x
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt-get install -y nodejs

# 验证
node -v
npm -v
```

### 步骤4：解压并部署

```bash
# 创建部署目录
mkdir -p /root/wordbook

# 解压文件
tar -xzf wordbook-deploy.tar.gz -C /root/wordbook

# 进入目录
cd /root/wordbook

# 查看文件结构
ls -la
```

### 步骤5：安装依赖

```bash
# 安装生产环境依赖
npm install --production

# 安装PM2进程管理器
npm install -g pm2
```

### 步骤6：配置防火墙

#### 查看当前防火墙状态：
```bash
# CentOS/RHEL
firewall-cmd --state

# Ubuntu
ufw status
```

#### 开放5000端口：

**CentOS/RHEL：**
```bash
firewall-cmd --permanent --add-port=5000/tcp
firewall-cmd --reload
```

**Ubuntu/Debian：**
```bash
ufw allow 5000/tcp
ufw reload
```

**重要：云主机安全组配置**
- 登录云主机控制台（阿里云/腾讯云/华为云等）
- 找到"安全组"或"防火墙"设置
- 添加入站规则：
  - 协议：TCP
  - 端口：5000
  - 来源：0.0.0.0/0（允许所有IP访问）

### 步骤7：启动服务

```bash
# 使用PM2启动（推荐）
pm2 start backend/server.js --name wordbook

# 查看运行状态
pm2 status

# 查看日志
pm2 logs wordbook

# 设置开机自启
pm2 startup
pm2 save
```

### 步骤8：访问测试

打开浏览器访问：
```
http://82.156.202.245:5000
```

应该能看到登录页面！

## 🔐 安全配置

### 1. 修改JWT密钥

编辑 `/root/wordbook/.env` 文件：
```bash
vi /root/wordbook/.env
```

修改 `JWT_SECRET` 为强密码：
```
JWT_SECRET=your-very-strong-secret-key-here-2026
```

然后重启服务：
```bash
pm2 restart wordbook
```

### 2. 定期备份数据库

```bash
# 创建备份目录
mkdir -p /root/wordbook-backups

# 备份数据库
cp /root/wordbook/data/wordbook.db /root/wordbook-backups/wordbook-$(date +%Y%m%d).db

# 查看备份
ls -lh /root/wordbook-backups/
```

### 3. 配置Nginx反向代理（可选）

```bash
# 安装Nginx
yum install -y nginx  # CentOS
# 或 apt install -y nginx  # Ubuntu

# 创建配置文件
vi /etc/nginx/conf.d/wordbook.conf
```

添加以下内容：
```nginx
server {
    listen 80;
    server_name 82.156.202.245;

    location / {
        proxy_pass http://localhost:5000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启动Nginx：
```bash
systemctl start nginx
systemctl enable nginx
```

然后可以通过 `http://82.156.202.245` 访问（不需要加端口号）

## 🐛 故障排查

### 问题1：服务无法启动

```bash
# 查看详细日志
pm2 logs wordbook --lines 100

# 常见错误：
# - 端口被占用：修改.env中的PORT
# - 权限不足：使用root用户
# - 依赖未安装：运行 npm install
```

### 问题2：无法访问网站

```bash
# 1. 检查服务是否运行
pm2 status

# 2. 检查端口是否监听
netstat -tlnp | grep 5000
# 或
ss -tlnp | grep 5000

# 3. 检查防火墙
firewall-cmd --list-ports  # CentOS
# 或
ufw status  # Ubuntu

# 4. 检查云主机安全组
# 登录云控制台确认5000端口已开放
```

### 问题3：数据库错误

```bash
# 检查数据库文件是否存在
ls -lh /root/wordbook/data/wordbook.db

# 如果数据库损坏，重新初始化
cd /root/wordbook
rm -f data/wordbook.db
pm2 restart wordbook
```

## 📈 性能优化

### 1. 设置PM2集群模式（多核CPU）

```bash
# 停止当前服务
pm2 stop wordbook

# 使用集群模式启动（根据CPU核心数）
pm2 start backend/server.js --name wordbook -i max

# 查看状态
pm2 status
```

### 2. 配置日志轮转

```bash
# 安装PM2日志轮转插件
pm2 install pm2-logrotate

# 配置
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 7
pm2 set pm2-logrotate:compress true
```
