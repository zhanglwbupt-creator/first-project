# 学习和复习功能测试指南

## 📋 功能概述

本次更新实现了完整的学习和复习功能，包括：

### ✅ 已实现功能

1. **学习模式选择** - 看词识意、拼写练习、复习单词
2. **看词识意** - 显示单词，从4个选项中选择正确释义
3. **拼写练习** - 显示释义，输入单词拼写
4. **艾宾浩斯复习** - 按1、2、4、7、15、30天间隔自动安排复习
5. **学习报告** - 显示学习统计、正确率、错题列表
6. **浏览器通知** - 自动设置复习提醒（默认晚上8点）
7. **每日学习计划** - 默认每天学习10个单词，可自定义

## 📖 测试流程

### 测试1：开始学习

1. **首页点击词库的"开始学习"按钮**
   - 应该跳转到学习模式选择页面
   - 显示学习统计（今日已学、累计学习、已掌握、正确率）

2. **选择学习模式**
   - 点击"📖 看词识意"或"✍️ 拼写练习"
   - 设置每日学习数量（默认10个）

3. **看词识意模式**
   - 显示单词和音标
   - 显示4个释义选项（A、B、C、D）
   - 点击选项后显示正确/错误反馈
   - 正确答案显示绿色，错误答案显示红色
   - 点击"下一个"继续

4. **拼写练习模式**
   - 显示中文释义和英文释义
   - 输入框中输入单词拼写
   - 按回车或点击"确认"
   - 显示正确/错误反馈
   - 错误时显示正确拼写

5. **完成学习**
   - 学完所有单词后自动跳转到学习报告
   - 或点击"结束"手动查看报告

---

### 测试2：学习报告

1. **查看统计数据**
   - 学习单词总数
   - 正确数量（绿色）
   - 错误数量（红色）
   - 正确率百分比

2. **查看正确率进度条**
   - 紫色渐变进度条显示正确率

3. **查看错题列表**
   - 如果有错误，显示需要加强的单词
   - 红色左边框高亮显示

4. **鼓励语**
   - 根据正确率显示不同鼓励语：
     - ≥90%: "🌟 太棒了！你是记忆大师！"
     - ≥70%: "👍 不错！继续保持！"
     - ≥50%: "💪 加油！还有进步空间！"
     - <50%: "📚 多多练习，一定会进步的！"

5. **操作按钮**
   - "继续学习" - 返回学习模式选择页面
   - "返回首页" - 回到首页

---

### 测试3：复习功能

1. **进入复习**
   - 从学习模式选择页面点击"🔄 复习单词"
   - 显示待复习单词数量

2. **复习流程**
   - 与看词识意类似
   - 显示当前复习阶段（第X次复习）
   - 红色主题头部（区别于学习）

3. **艾宾浩斯算法**
   - 第1次复习：1天后
   - 第2次复习：2天后
   - 第3次复习：4天后
   - 第4次复习：7天后
   - 第5次复习：15天后
   - 第6次复习：30天后

4. **复习完成**
   - 同样显示学习报告
   - 记录复习结果

---

### 测试4：浏览器通知

1. **首次访问**
   - 浏览器会弹出通知权限请求
   - 点击"允许"

2. **通知设置**
   - 控制台显示：`复习提醒已设置: [日期时间]`
   - 默认每天晚上8点提醒

3. **通知效果**
   - 到达设定时间后弹出浏览器通知
   - 标题：📚 背单词提醒
   - 内容：有X个单词待复习，打开应用开始复习吧！
   - 点击通知会聚焦到应用窗口

---

### 测试5：数据持久化

1. **学习记录保存**
   - 每次学习/复习后自动保存到数据库
   - 刷新页面后统计数据仍然保留

2. **每日限制**
   - 每天学习相同数量的单词（默认10个）
   - 已学习的单词今天不会再出现
   - 第二天可以继续学习新单词

3. **复习计划**
   - 根据正确/错误自动计算下次复习日期
   - 正确的单词进入下一阶段
   - 错误的单词重置到第一阶段

## 🎯 关键API端点

```bash

# 获取待复习单词（20个）

  "bankId": 1,
  "wordId": 123,
  "studyType": "learn",  // learn 或 review
  "correct": true
}

# 获取学习统计
GET http://localhost:5000/api/study/stats/1
```

## 📊 数据库表结构

### study_records 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 记录ID（主键） |
| word_id | INT | 单词ID |
| bank_id | INT | 词库ID |
| study_date | DATE | 学习日期 |
| study_type | TEXT | learn/review |
| status | INT | 0-未学 1-学习中 2-已掌握 |
| correct | INT | 0-错误 1-正确 |
| review_stage | INT | 复习阶段 0-5 |
| next_review_date | DATE | 下次复习日期 |

## ⚠️ 注意事项

1. **浏览器通知权限**
   - 需要用户手动允许
   - iOS Safari支持有限
   - 建议使用Chrome/Edge测试

2. **学习进度**
   - 数据保存在服务器数据库
   - 清除数据库会丢失学习记录

3. **复习算法**
   - 基于艾宾浩斯遗忘曲线
   - 错误单词会重置复习阶段
   - 正确单词进入下一阶段

4. **每日限制**
   - 按日期判断是否已学习
   - 跨天后可以学习新单词
   - 复习不受每日限制

## 🐛 常见问题

### Q1: 学习后统计数据不更新？
A: 检查后端API是否正常返回，查看浏览器控制台错误

### Q2: 没有待学习单词？
A: 确保词库中有单词，且今天还没有学习过

### Q3: 复习单词数量为0？
A: 需要先学习单词，学习后根据复习阶段安排复习时间

### Q4: 通知没有弹出？
A: 
1. 检查是否允许了通知权限
2. 检查系统时间是否到达设定时间
3. 查看控制台是否有错误
